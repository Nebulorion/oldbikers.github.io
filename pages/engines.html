Движки | На чём пишут игры?

Любой, кто планировал создание своей компьютерной игры, задумывался о том, на чём она будет реализована. Не будет секретом, что для создания игры можно использовать любой из доступных инструментов, как для написания кода и создания ПО. При желании игру можно создать даже в редакторе электронных таблиц Excel.

3D движок на базе Microsoft Excel

Конечно же нас интересуют наиболее удобные и продвинутые решения для создания игры и об этом был один абзац на первом шаге RoadMap, который к тому же предлагался в качестве одного из аргументов об актуальности разработки своей компьютерной игры:

«…с технической точки зрения, для создания игры можно использовать библиотеки к определённым ЯП (языкам программирования), которые, например, могут облегчить работу с объектами в 3D пространстве. Иначе существуют целые так называемые движки, которые уже обладают всем необходимым инструментарием- от графического отображения уровня до непосредственно логики игры. Если не стоит задачи разработать новый концепт, а лишь добавить что-либо на базе существующего, то для таких целей вполне можно использовать API готовых игр для создания своих модификаций. Модификации, если они предусмотрены разработчиком созданной игры, могут быть созданы при помощи внедрённого скриптинга (Lua, Kotlin, Python, и т.д.), библиотеки с API для ЯП (С++, С#, Java, и т.д.) или специального редактора, для работы с которым знания ЯП может быть необязательным».

Почему же чаще обращаются к готовым движкам?

Под игровым движком чаще всего подразумевается такое программное обеспечение, которое обладает всем необходимое для создания игры. Такие движки создаются и для личного пользования (создания своей игры) и на продажу для других разработчиков (чаще всего это делают компании, которые уже разработали игру на специально разработанном для неё движке, доработанным позже для большего охвата возможных задач). Работать с готовыми движками удобнее, т.к. за их производство отвечают компании, специализирующиеся именно на производстве такого продукта. Также по ним можно найти больше информации в интернете- документацию разработчика, форумы пользователей продукта, примеры созданных на этих движках игр.

Классификации игровых движков

Игровые движки можно разделить по игровым жанрам:

  1. Узконаправленные (непредназначенные для других жанров; например- RPG Maker).
    RPG Maker- программа для создания игр жанра JRPG (источник)
  2. Широкого пользования (поддерживающие гибридные жанры или подходящие для любого из жанров).
    CryEngine - игровой движок(источник)

Достаточно часто движки делят на ориентированные для создания 2D или 3D игр. Стоит отметить, что 3D движки также позволяют создавать и 2D игры, а 2D движки могут внедрять 3-е измерение для визуализации, а не игровой механики.

  1. Среди 2D движков можно выделить Construct 2, GameMaker: Studio, RPG Maker, Cocos2d.
    GameMaker: Studio- движок для создания 2D игр (источник)
  2. Среди 3D движков- Source 2, CryENGINE 3, Unreal Engine 4, Unity 5, Godot 3. Движок Source от Valve получил известность благодаря играм Counter-Strike: Source и Half-Life 2. CryEGNINE давно себя зарекомендовал среди разработчиков, ориентированных на создание AAA-игр. Godot- движок, только набирающий популярность.
    Godot - набирающий популярность движок для создания 3D игр

Unreal Engine и Unity

Самыми востребованными среди разработчиков игр являются 2 движка:

Unreal Engine

Unreal Engine - игровой движок с продолжительной историей. Он давно был известен своими возможностями, но получил популярность, как только стал полностью бесплатным (исключением являются 5% с продаж за преодоление $1,000,000 USD монетизации игрового продукта). Дополнительную известность движок получил благодаря его использованию на съёмках сериала «Мандалорец». Его сильные стороны: реалистичная графика, широкий инструментарий и удобство реализации (редакторы анимации и поддержка визуального программирования почти в любых областях разработки). Unreal Engine поддерживает скрипты визуального программирования BluePrints, но это не исключает возможности написания кода на C++. Считается, что у Unreal Engine более высокие требования к ресурсам компьютера (и, несмотря на кроссплатферменность, могут появляться сложности в поддержке на мобильных устройствах без должной оптимизации) и до недавних пор наличие малого количества бесплатных библиотек и документации по движку. В недавние время Epic Games стала создавать много качественных бесплатных библиотек и материалов благодаря покупке компании Quixel.

Unreal Engine 5 - один из самых ожидаемых игровых движков

Unity3D

Unity3D- наиболее доступный игровой движок. Его основным преимуществом являются: лёгкость в разработке (код на C# и JS, есть платная поддержка визуального программирования, из-за чего оно менее популярно, чем на UE4), распространяется бесплатно (но есть платные версии движка), высокая популярность для производства игр на VR, AR и мобильных устройствах. По Unity документации и учебного материала и контента значительно больше, чем у любого другого движка, как и библиотек (но это не означает, что качество не оставляет желать лучшего).

Unity 5 - самый популярный игровой движок

В заключение

Движок- это самый важный инструмент для разработчика компьютерных игр. Писать свой игровой движок, как этот делал для Minecraft Маркус Персон, сейчас нет необходимости. Выбор готового движка следует из задач, которые он должен решать, но в большинстве случаев выбор будет стоять только между Unreal Engine и Unity.
